What is an N-gram?

An N-gram is just a fancy name for a group of words that appear together in a sentence.

Think of it like this: If you’re reading a sentence, an N-gram looks at N words in a row.

For example:

Let’s take this sentence: “I love chocolate cake.”

Here’s how we break it into different N-grams:

  • 1-gram (unigram) → just one word at a time: “I”, “love”, “chocolate”, “cake”

  • 2-gram (bigram) → two words in a row: “I love”, “love chocolate”, “chocolate cake”

  • 3-gram (trigram) → three words in a row: “I love chocolate”, “love chocolate cake”

  • 4-gram → four words in a row: “I love chocolate cake”

Why do computers use N-grams?

Computers don’t understand sentences like we do, so they learn patterns by looking at which words often appear together.

If you often say “peanut butter and…”, the computer will learn that “jelly” is a good guess for the next word!

That’s what N-grams are all about—helping computers guess what word comes next by learning from lots and lots of word pairs (or triplets, or quadruplets…).

So, in short: An N-gram is a small group of N words in a row that helps computers understand how words usually go together.

  2. How many times was each word used in the verse? For example, the word “What” was used only once but the word “more” was used twice.

What do you think? Earlier, n-grams were explained as a way for a computer to predict the next words. Do you think unigrams will tell us what the next word will be in a sentence? Perhaps not. Then, let’s try bigrams.

Let’s try bigrams!

# Step 1: Load required package
# library(dplyr)

# Step 2: Create bigrams
bigrams <- tibble(
  w1 = head(tokens, -1),
  w2 = tail(tokens, -1)
)

# Step 3: Count bigrams
bigram_counts <- bigrams |> 
  count(w1, w2, name = "bigram_count")

# Step 4: Count unigrams (for denominator)
unigram_counts <- tibble(word = tokens) |> 
  count(word, name = "unigram_count")

# Step 5: Join and compute bigram probabilities
bigram_probs <- bigram_counts |> 
  left_join(unigram_counts, by = c("w1" = "word")) |> 
  mutate(probability = round(bigram_count / unigram_count, 4)) |> 
  arrange(desc(probability))

# Step 6: View the result
print(bigram_probs, n = Inf)

Let’s generate sentences out of the bigram model above.

# Example: generate a sentence using bigram model
generate_sentence <- function(bigram_probs, start_word = "what", max_len = 20) {
  sentence <- c(start_word)
  
  current_word <- start_word
  for (i in 1:max_len) {
    # Filter possible next words
    candidates <- bigram_probs[bigram_probs$w1 == current_word, ]
    if (nrow(candidates) == 0) break  # stop if no continuation
    
    # Sample next word according to probability
    next_word <- sample(candidates$w2, size=1, prob=candidates$probability)
    
    sentence <- c(sentence, next_word)
    current_word <- next_word
  }
  paste(sentence, collapse=" ")
}

# Example usage:
set.seed(123)
generate_sentence(bigram_probs, start_word="what")
